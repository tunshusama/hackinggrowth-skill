# Archetypes and Cases

Use this file after choosing one product archetype.

Important:
- When citing examples, use book examples only.
- Prefer these canonical cases from Hacking Growth: Airbnb, Dropbox, Facebook, LinkedIn, Yelp.

## 1) Marketplace

Core constraint:
- Liquidity and trust on both sides

Prioritize:
- Supply quality and response speed
- Demand intent matching
- Trust systems (reviews, identity, guarantees)
- Double-sided activation metrics

Book examples:
- Airbnb: trust, listing quality, and demand loops
- Yelp: review flywheel and local discovery quality

Starter experiment ideas:
- New supply onboarding checklist with quality score
- Fast-response badge for top suppliers
- Demand-side recommendation and ranking experiments

## 2) Social / Content Platform

Core constraint:
- Content density and repeat engagement loops

Prioritize:
- First follow graph quality
- Feed relevance and posting frequency
- Notifications/lifecycle triggers tied to value moments
- Sharing loops and creator incentives

Book examples:
- Facebook: identity graph and invite mechanics
- LinkedIn: profile completion and network-density activation

Starter experiment ideas:
- First-session interest picker to improve feed cold start
- New user follow recommendations for first-week relevance
- Share prompt at peak delight moment (not at random)

## 3) PLG SaaS

Core constraint:
- Time-to-first-value and team expansion

Prioritize:
- Template-led onboarding
- Guided setup to first successful output
- Collaboration invites right after first value
- Upgrade path tied to usage thresholds

Book examples:
- Dropbox: referral loops and habit reinforcement
- LinkedIn: activation via completing core setup steps

Starter experiment ideas:
- 3-step setup wizard with sample data
- "First win in 10 minutes" onboarding variant
- Invite teammate prompt after first successful workflow

## 4) Ecommerce

Core constraint:
- Conversion friction and repeat purchase

Prioritize:
- Landing-to-product page relevance
- Cart and checkout drop-off reduction
- Merchandising and recommendation quality
- Retention via replenishment and post-purchase moments

Book examples:
- Use Dropbox/Facebook-style loop thinking for conversion and retention mechanics
- Adapt Airbnb/Yelp trust patterns for social proof and credibility cues

Starter experiment ideas:
- Product page social proof placement test
- Checkout step reduction with autofill defaults
- Post-purchase recommendation flow for second order

## 5) Developer Tool

Core constraint:
- Time-to-hello-world and integration reliability

Prioritize:
- Docs entry clarity
- Quickstart completion rate
- SDK/API error recovery
- Activation tied to first successful API call

Book examples:
- Use Dropbox first-value acceleration patterns
- Use LinkedIn onboarding-completion patterns

Starter experiment ideas:
- Language-specific quickstart selector on docs homepage
- Copy-paste starter project with one-command run
- Error-state remediation guides inline in dashboard
